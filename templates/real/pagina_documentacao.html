Painel Técnico

Arquitetura do Sistema

O Participa DF Privacy Shield foi projetado sob o paradigma de microsserviços lógicos, garantindo que o motor de análise seja desacoplado da interface de usuário.

  • Core Engine: Módulo Python especializado em processamento de strings e validação de PII.
  • ORM Layer: Utilização do SQLAlchemy para abstração do banco de dados (SQLite/PostgreSQL).
  • Pipeline de Validação: Fluxo sequencial: Sanitização → Regex → Checksum → NLP.

Guia de Instalação

Configuração do ambiente (Python 3.10+):

# Clonar o repositório oficial
git clone https://github.com/CalebeAF02/Gerenciador-De-Hoteis---UNB.git

# Criar e ativar o ambiente virtual
python -m venv venv
source venv/bin/activate

# Instalar dependências
pip install -r requirements.txt

Execução do Projeto

Inicie o servidor de desenvolvimento:

python app.py

Acesse em: http://127.0.0.1:5000

Metodologia Híbrida

1. Regex de Alta Performance

Identificação de padrões estruturais (E-mails, CEP, Telefones).

2. Validação Matemática (Checksum)

Algoritmo de Módulo 11 para CPF e CNPJ, eliminando falsos positivos.

3. NLP (Reconhecimento de Entidades)

Detecção de nomes e cargos via Processamento de Linguagem Natural.

Segurança e LGPD

Conformidade técnica com a legislação:

  • Minimização: Dados brutos não são persistidos permanentemente.
  • Sanitização: Proteção contra XSS e SQL Injection.
  • Hashing: Logs de auditoria gerados com SHA-256.
